Google Sign-In (OAuth Client ID)
Needed to view your subscriptions & liked videos. Free to set up.
✓ FREE · Same Google Cloud project
  1. 1
    In your Google Cloud project
    Go to APIs & Services → Credentials.
  2. 2
    Create OAuth 2.0 Client ID
    Create Credentials → OAuth client ID → Application type: Web application.
  3. 3
    Add your hosted URL
    Under "Authorized JavaScript origins" add the URL where ViewTube is hosted (e.g. your GitHub Pages URL).
  4. 4
    Copy Client ID
    Click Create → copy the Client ID (ends in .apps.googleusercontent.com).
  5. 5
    Also enable YouTube Data API v3
    Make sure the same project has YouTube Data API v3 enabled (same as API Key setup).
Note: Sign-in requires ViewTube to be served from a real URL (not a local file). See the Hosting Guide below.
Hosting & Google Sites Embed
To embed ViewTube in Google Sites, you must host it at a public HTTPS URL first.
✓ FREE · GitHub Pages
  1. 1
    Create a free GitHub account
    Visit github.com and sign up (free).
  2. 2
    Create a new repository
    Click + → New repository → name it viewtube → Public → Create.
  3. 3
    Upload the HTML file
    Click "uploading an existing file" → drag viewtube.html → rename it to index.html → Commit.
  4. 4
    Enable GitHub Pages
    Repository Settings → Pages → Branch: main → Save. Your URL will be: yourusername.github.io/viewtube
  5. 5
    Embed in Google Sites
    In your Google Site → Insert → Embed → By URL → paste your GitHub Pages URL → Insert.
  6. 6
    Add URL to OAuth origins
    In Google Cloud Console → Credentials → your OAuth client → add your GitHub Pages URL to Authorized JavaScript origins.
Why can't I just use the HTML file directly in Google Sites? Google Sites requires a live HTTPS URL. A downloaded .html file has no URL — it only exists on your computer.
